## Renamed Core Concepts

The three main things you work with in WirkSam have been renamed to match how people actually talk about the system. No behavior has changed — every button, page, and action works the same as before, just with clearer labels.

| Before       | Now       | What it means                                                  |
| ------------ | --------- | -------------------------------------------------------------- |
| Event Group  | **Event** | A time-bounded container like a camp, festival, or season      |
| Event        | **Task**  | A specific unit of work inside an Event (e.g. "Run kitchen")   |
| Duty Slot    | **Shift** | A bookable time block within a Task ("Saturday 14:00–16:00")   |

The old words were ambiguous ("Event" and "Event Group" sounded almost the same) and one was technical jargon ("Duty Slot"). The new vocabulary reads naturally in both English and German — Veranstaltung, Aufgabe, Schicht.

## What You'll Notice

- Sidebar and navigation labels use the new terms throughout.
- URLs have changed: `/events` now lists your tasks, `/event-groups` is gone (use `/events` with the new meaning), `/duty-slots` is now `/shifts`. Existing bookmarks to the old paths will need to be updated.
- Notifications are titled with the new names ("Task Published", "Shift Starting Soon", etc.).
- All your existing events, tasks, shifts, bookings, and availability entries were preserved — only the names on top changed.

## Under the Hood

The rename was applied as a single coordinated update across the backend, the database, the frontend, and the generated API client. The database migration also merges any duplicate notification types that the seeder may have created alongside the old ones, so no manual cleanup is required.
